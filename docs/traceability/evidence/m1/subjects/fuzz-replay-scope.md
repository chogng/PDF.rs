# M1 document-service fuzz and minimizer scope

The registered `m1documentservices` target is a real `libfuzzer_sys::fuzz_target!` that constructs `OpenStrictBaseRevisionJob`, drives `M1StrictDocumentSession` through exact Range responses, and, when strict opening reaches Ready, requests both Native page-count and outline services. Its invariant is that every byte string stays within the declared input, timeout, RSS, and session budgets without panic or resource escape.

The gating test pins `cargo-fuzz 0.13.2`, whose crates.io source package is bound to `sha256:5acfd01930e49823e58c30dd8012d3338a620377d7c7d4cc140ca4b2169400e2` and is provisioned with `cargo install --locked --version 0.13.2 cargo-fuzz`. It compiles the nested package with `cargo check --locked` and launches `cargo fuzz run` with `--sanitizer none`, target `m1documentservices`, exactly one temporary corpus argument, `-seed=424242`, `-runs=64`, `-max_len=1048576`, `-timeout=1`, and `-rss_limit_mb=512`. The nested manifest directly pins `libfuzzer-sys 0.4.13`; its complete transitive closure is bound by the nested Cargo.lock. The content-addressed 25-entry PDF dictionary is supplied to libFuzzer.

A separate test launches `cargo fuzz cmin` with the same target, seed, input bound, timeout, RSS bound, and dictionary. It verifies that cmin replaces the three reviewed seed filenames—minimal, truncated-header, and valid nested-outline—with a nonempty, nonexpanding coverage-selected corpus; this run retained three outputs. `cargo fuzz tmin` is not applicable because the bounded campaign produced no crash artifact; the evidence records that explicitly instead of claiming crash minimization.

All registered replay/cmin corpus and crash-artifact writes occur in unique temporary directories and are removed by the test. Cargo-fuzz build products and its ignored working directories remain non-product local tooling and are not registered evidence. Both fuzz dependencies are separately recorded as conditional non-product test/toolchain inputs in the dependency ledger and remain outside the product closure. This is deterministic gating coverage evidence, not a nightly-duration or CANARY-scale campaign, and it does not involve PDFium or any O4 observation.
